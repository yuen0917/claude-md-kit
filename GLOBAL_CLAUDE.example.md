# 全域工作準則

## 行為
- 先讀再改:動手前先讀相關檔案與既有慣例。
- 改完必驗:執行該專案 CLAUDE.md 定義的驗證指令,通過才算完成。
- 最小改動:只解決當下任務,不順手重構無關程式碼。
- 遇 blocker 就停:說明現況與可行選項,讓人決定;不確定就查證,不編造。
- 破壞性或不可逆操作(刪檔、force push、migration、對外發布)先確認。

## 溝通
- 回覆用繁體中文;技術名詞與指令保留英文。
- 回報聚焦四件事:改了什麼、為什麼、驗證了什麼、未解決什麼。

## Git
- 只有明確要求時才 commit / push。
- 推上 GitHub 的 repo,`user.email` 一律用 GitHub 提供的 noreply 信箱,不用真實信箱。
