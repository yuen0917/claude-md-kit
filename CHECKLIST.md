# 套用後自我檢核清單

複製範本、填完內容後,逐項確認;任何一項不過就回頭修。

## 長度與密度
- [ ] 專案根目錄 CLAUDE.md 在 60–80 行以內(超過會開始被 agent 忽略)
- [ ] 每一行都通過「刪掉這行,agent 會不會出錯?」的測試
- [ ] IMPORTANT / YOU MUST / 絕不 等強標記,全檔合計 ≤ 3 處
- [ ] 範本的填寫註解(`<!-- -->`)已全部刪除

## 內容取捨
- [ ] 驗證指令實際跑過一次,可直接複製執行
- [ ] 沒有人格指令(「當個資深工程師」「一步步思考」)
- [ ] 沒有 formatter / linter 已能強制的格式規則(那些改用 hook)
- [ ] 沒有從目錄結構一看即知、無法轉為行動的資訊
- [ ] 沒有與全域 `~/.claude/CLAUDE.md` 重複的通用準則

## 分層
- [ ] 子系統專屬規則放子目錄 CLAUDE.md,而非塞進根目錄
- [ ] 檔案類型綁定的規則放 `.claude/rules/` 並附 glob
- [ ] 大型文件放 `docs/`,CLAUDE.md 只寫「何時去讀哪份」,沒有 `@` 全文嵌入

## 狀態與安全
- [ ] `docs/STATE.md` 已建立,CLAUDE.md 有指明讀寫時機
- [ ] 沒有任何秘密出現在 CLAUDE.md / rules / docs / 範例程式碼
- [ ] `.env` 等秘密檔已列入 `.gitignore`
- [ ] 啟用中的 MCP 為精選(整體佔用 < 約 20k token);能用 Skill 的已改用 Skill
